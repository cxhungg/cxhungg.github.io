Network Traffic Anomaly Detection

07/09/2025

I did this project to compare the effectiveness of anomaly detection in network traffic between supervised and unsupervised learning. The dataset used is CICIDS2017, a dataset from the Canadian Institute for Cybersecurity.

Supervised Learning vs. Unsupervised Learning

For supervised learning, I decided to use Random Forest because it can handle non-linear relationships and is relatively easy to implement. XGBoost is another supervised learning model that uses decision trees but in a different way, and is apparently slightly better than Random Forest. XGBoost requires more hyperparameter tuning, but I didn't do that in this project because it took too long to run when I tried.

For unsupervised learning, I decided to use Isolation Forest because it is made for anomaly detection. Instead of learning a profile of normal data, it tries to isolate anomalies. I used Autoencoder because I learned about it in one of my recent classes and I wanted to try it out.

Summary

  • Developed and compared supervised (Random Forest, XGBoost) and unsupervised (Isolation Forest, Autoencoder) models to detect anomalies in network traffic using the CICIDS2017 dataset.
  • Achieved 96.69% AUC with XGBoost and 87.14% AUC with Autoencoder by engineering flow-based network features and performing time-series traffic analysis.
  • Visualized model performance using ROC curves, confusion matrices, and t-SNE clustering.

Used: Python, Scikit-learn, XGBoost, TensorFlow, Pandas, NumPy, Matplotlib, Seaborn
